Hexo 本地项目构建

什么是 Hexo ？

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装 hexo (需要先安装 nodejs/npm)

初始化项目(建站)

解释:

  • 第一行hexo init命令, 将会创建一个项目, 参数<folder>即项目名称, 也就是项目根目录文件夹的名称,hexo init <folder>需要将<folder>更换为自己的项目名称（别傻呼呼不知道自己的项目名称是什么, 就是自己取的名字）
  • 第二行cd命令, 意思为进入到文件夹内, <folder>应该和hexo init <folder> 中的 <folder> 一致, 在我们执行完第一行的时候, 应该在当前目录下生成一个这样的文件夹, 以及 hexo 初始文件
  • 第三行npm install意思为安装该项目相关依赖.

目录结构说明

新建完成后，指定文件夹的目录如下:

  • _config.yml, 网站的 配置 信息，您可以在此配置大部分的参数.
  • package.json, 应用程序的信息。
  • scaffolds, 模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
  • source, 资源文件夹是存放用户资源的地方。
  • themes, 主题 文件夹。Hexo 会根据主题来生成静态页面。

本地启动(运行)

解释:

  • 启动服务器, 默认情况下, 访问网址为: http://localhost:4000

本篇总结

本篇一共就四条核心命令, 走出了我们从0到1的第一步, 启动服务器之后, 通过访问本地4000端口, 可以访问一个默认主题的hexo博客

更多详细的命令使用说明请参考 hexo官方文档, 下篇将讲述 hexo 主题的应用.
